Command Injection Vulnerability

Concept

Command injection happens when user input is passed directly to system commands.

Example vulnerable code:

os.system("ping " + user_input)

Attackers can append additional commands to the input.

Example Attack

Show example input:

8.8.8.8; cat flag.txt

The semicolon allows execution of another command.

Real World Impact

Attackers could run commands like:

ls
whoami
cat /etc/passwd

This could lead to full server compromise.

Prevention

Best practices:

  • Never pass raw user input to shell commands
  • Use safe libraries instead of shell execution
  • Sanitize and validate input
