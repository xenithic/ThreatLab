Broken Authentication

Concept

Applications sometimes trust user input for authentication instead of securely verifying identity on the server side.

If an application relies on client-supplied data (like a URL parameter or form field) to determine who is logged in, an attacker can simply change that value.

Example

A URL like:

/profile?user=admin

could grant unauthorized access to the admin profile if the server blindly trusts the user parameter.

Impact

Attackers can impersonate other users, access privileged accounts, and perform unauthorized actions across the platform.
