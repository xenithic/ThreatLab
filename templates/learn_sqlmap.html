SQL Injection using sqlmap

Concept

SQL Injection is a web vulnerability that allows attackers to interfere with database queries by injecting malicious SQL code through user input.

When user input is directly concatenated into an SQL query without sanitization, attackers can manipulate the query to:

  • Bypass authentication
  • Read sensitive data from the database
  • Modify or delete data
  • Execute administrative operations

Vulnerable Code Example

Consider this backend query:

query = "SELECT * FROM products WHERE id = " + user_input

If the user supplies 1 OR 1=1, the query becomes:

SELECT * FROM products WHERE id = 1 OR 1=1

This returns all rows from the table because 1=1 is always true.

What is sqlmap?

sqlmap is an open-source penetration testing tool that automates the detection and exploitation of SQL injection vulnerabilities.

It can:

  • Detect injection points automatically
  • Enumerate databases, tables, and columns
  • Dump database contents
  • Attempt OS-level access in severe cases

Example Commands

Basic detection scan:

sqlmap -u "http://{{ target_host }}:5000/search?id=1"

Dump all data:

sqlmap -u "http://{{ target_host }}:5000/search?id=1" --dump

Types of SQL Injection

Common Techniques:

Tautology:    1 OR 1=1           → returns all rows
UNION-based:  1 UNION SELECT ...  → extracts data from other tables
Error-based:  1'                  → triggers error revealing DB info
Blind SQLi:   1 AND 1=1           → true/false responses leak data

Real World Impact

SQL injection has been responsible for some of the largest data breaches in history, exposing millions of user credentials, payment details, and personal information.
