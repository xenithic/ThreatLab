Brute Force Attack using Hydra

Concept

Hydra is a fast and flexible online password cracking tool. It supports numerous protocols and can be used to brute-force login forms, SSH, FTP, and many other services.

Brute-force attacks work by systematically trying every password from a wordlist until the correct one is found.

How Hydra Works

Hydra sends login attempts to a target and checks the response. If the response does not contain a specific failure message, it assumes the credentials are correct.

  • -l — specify the username
  • -P — specify the password wordlist file
  • http-post-form — attack type for web login forms

Example Command

hydra -l admin -P passwords.txt -s 5000 {{ target_host }} http-post-form "/hydra-login:username=^USER^&password=^PASS^:Invalid credentials"

Breaking it down:

Command Breakdown:

-l admin                → target username
-P passwords.txt        → wordlist of passwords to try
{{ target_host }}               → target host
http-post-form          → attack a web form via POST
"/hydra-login            → form action URL
:username=^USER^         → username field (^USER^ is replaced by -l value)
&password=^PASS^        → password field (^PASS^ is replaced by wordlist entries)
:Invalid credentials"    → failure string (if response contains this, login failed)

Real World Impact

Weak passwords are one of the most common attack vectors. Brute-force tools like Hydra can compromise accounts that use simple or commonly known passwords within seconds.
